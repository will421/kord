CacheEntitySupplier

EntitySupplier that uses a DataCache to resolve entities.Getting existing entities should not throw any RequestException unless specified by the configured cache.The supplier might not be able to return entities independent of their actual existence, their presence depends on incoming events from the Gateway and how the cache is set up to store entities.Returned flows without entities will not throw an EntityNotFoundException if none are presented like other getX functions. Instead, the flow will be empty.
